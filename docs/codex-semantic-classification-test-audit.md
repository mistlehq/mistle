## Recommended Follow-up Tests

Highest priority remaining gaps:

1. Add explicit reducer hydration coverage for grouped `making-edits`, `searching-web`, `tool-call`, and `running-commands` adjacency if we want full acceptance-checklist parity at the dashboard layer.
2. Add a dedicated reducer regression showing `user-message` also acts as a hard grouping boundary, so the final standalone-boundary acceptance item can move from `Partial` to `Covered`.
3. Add a targeted reducer ordering regression around grouped semantic blocks and standalone entries if we want to move transcript-order preservation from `Partial` to `Covered`.
